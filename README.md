# 电子花园 Web 项目（Electronic Garden）

这是一个用于构建「电子花园」网页系统的项目。

这个网页把**社区花园的日常运作**（照料、排班、协作、激励、治理）  
设计成一个**像玩农场游戏一样的可视化工作台**，让参与者更容易理解状态、完成任务、参与决策。

---

## 这个网页最终包含什么内容？

从用户视角看，这个网页主要由 **4 个页面（Tab）** 组成：

1. 🌿 **我的花园**  
   - 花园地图 / 床位状态  
   - 作物生长进度、是否缺水、是否成熟  
   - 点击床位进行管理或求助

2. 📋 **任务与协作**  
   - 我的排班任务  
   - 逾期任务与协作大厅  
   - 认领任务、代班、求助

3. 🏆 **社区与激励**  
   - 积分与等级  
   - 排行榜  
   - 礼物 / 奖励 / 成就

4. ⚖️ **治理与台账**  
   - 提案与投票  
   - 财务台账  
   - 培训与认证

> 本仓库的目标不是一次性把这四个页面全部做完，  
> 而是**先搭好一个清晰、可扩展的代码框架**，再逐步把内容填进去。

---

## 这个项目的代码是如何组织的？（一句话版）

- **页面负责“放哪些内容”**
- **模块负责“长什么样、怎么交互”**
- **数据和规则集中管理**
- **新增功能尽量只加文件，不改旧文件**

## src 目录：你真正会改代码的地方

`src/` 是整个项目的核心。  
**几乎所有功能修改、新功能添加，都会发生在这里。**

```text
src/
├── pages/        # 页面入口（每个页面/Tab 对应一个文件）
├── sections/     # 页面上的“功能模块”（最常新增）
├── core/         # 项目骨架（模块如何组合、如何交互）
├── services/     # 数据从哪里来（后端 / 云函数）
├── modules/      # 业务规则（不管 UI）
├── components/   # 通用 UI 组件
├── styles/       # 全局样式
└── assets/       # 图片 / 插画 / 图标

## 页面内容是在哪里实现的？

这个项目里，“页面”本身非常轻，它不负责画复杂 UI。

页面只做三件事：
1. 决定这是哪一个页面（花园 / 任务 / 社区 / 治理）
2. 决定页面上要显示哪些模块（sections）
3. 把模块组合并显示出来

```text
src/pages/
├── Garden.tsx        # 🌿 我的花园 页面
├── Tasks.tsx         # 📋 任务与协作 页面
├── Community.tsx     # 🏆 社区与激励 页面
└── Governance.tsx    # ⚖️ 治理与台账 页面

## sections：最重要、最常新增的地方

`sections/` 是这个项目里**最重要、也最常改动的目录**。

这里的每一个文件夹，代表页面上的一个「功能模块」。

```text
src/sections/
├── banner/            # 顶部提示（天气 / 公告 / 建议）
├── plotCardList/      # 花园床位模块
├── todoList/          # 任务 / 待办模块
├── quickActions/      # 快捷操作模块
└── （以后新增的模块）
