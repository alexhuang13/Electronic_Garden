# 电子花园 Web 项目

这是一个用于构建「电子花园」网页端系统的代码仓库。

这个项目的目标不是一次性做完所有功能，而是**先搭建一个清晰、可扩展的框架**，让后续新增功能（比如花园管理、任务协作、积分激励、社区治理）都能**像“拼积木”一样加入**，而不需要推倒重来。

---

## 这个项目在做什么？

简单来说，这是一个 **“日常使用的管理型网页”**：

- 打开网页，可以快速看到当前花园/任务/社区的整体状态
- 页面会引导用户完成今天该做的事情
- 不同功能以“模块”的形式展示，可以灵活组合、调整顺序、随时扩展

它更像一个 **工作台（Dashboard）**，而不是普通的信息展示网页。

---

## 项目设计的核心思想

### 1️⃣ 页面由“模块”组成，而不是写死的布局

页面不是写死的 HTML 结构，而是由一组模块（Sections）组成：

- 顶部提示模块（天气 / 公告）
- 花园床位卡片模块
- 待办任务模块
- 快捷操作模块
- ……

每个模块都是一个独立的小组件，页面只是把它们“摆出来”。

这样做的好处是：
- 新增功能 = 新增一个模块
- 不需要改原有页面结构
- 模块可以随时插入、删除或调整顺序

---

### 2️⃣ 代码结构清晰，方便不同角色的人协作

这个仓库的结构希望做到：
- **新手**：能快速看懂项目在干什么
- **前端**：知道 UI 在哪里写
- **后端/云函数**：知道数据从哪里来
- **未来的你**：半年后回来还能理解

---

## 如何阅读这个仓库？

如果你是第一次打开这个项目，推荐按下面顺序阅读：

1. 先看 **README.md（本文件）**
2. 再看 `src/pages/` —— 每个页面的入口
3. 然后看 `src/sections/` —— 页面上每个模块的实现
4. 再看 `src/core/` —— 项目的“骨架逻辑”
5. 最后再看 `services/`、`modules/` 等细节
